# Mnemosyne / KnowCran 1.0.0 Release Checklist

Use this checklist before tagging `v1.0.0`.

## Project Metadata

- [ ] `pyproject.toml` version is `1.0.0`.
- [ ] `knowcran.__version__` is `1.0.0`.
- [ ] PyPI development classifier is intentionally set for the release state (`4 - Beta` for RC, `5 - Production/Stable` only after all gates pass).
- [ ] `README.md` describes current behavior and limitations.
- [ ] `LICENSE` exists and matches `pyproject.toml`.
- [ ] `CHANGELOG.md` has a 1.0.0 entry.
- [ ] `CONTRIBUTING.md` describes local development and quality gates.
- [ ] `ROADMAP.md` describes post-1.0 scope.

## Verification

- [ ] `pytest -v`
- [ ] `pytest --cov=knowcran --cov-report=term-missing`
- [ ] `python -m compileall knowcran tests`
- [ ] `python -m build`
- [ ] `bash scripts/verify-release.sh` or `.\scripts\verify-release.ps1`
- [ ] If using `uv`, refresh and verify `uv.lock` after dependency changes.
- [ ] GitHub Actions passes on Linux, macOS, and Windows for Python 3.12 and 3.13.
- [ ] Release workflow smoke-installs the built wheel and verifies both `knowcran` and `mnemosyne` entry points.
- [ ] Package smoke job verifies `knowcran --help`, `knowcran doctor`, and local service module imports without model downloads.

## Local Managed Services & GPU Verification

- [ ] `knowcran doctor --gpu` executes and successfully checks CUDA, torch availability, and Docker.
- [ ] `knowcran services start --gpu` boots uvicorn and compose configurations correctly.
- [ ] `knowcran services status` shows correct statuses, ports, PIDs, and backend information.
- [ ] `knowcran services logs embedding` and `knowcran services logs mineru` correctly print service logs.
- [ ] `knowcran services stop` gracefully terminates uvicorn and runs compose down.
- [ ] Automated tests in `tests/test_services.py` pass.
- [ ] In a prepared local/GPU environment, the local embedding server health endpoint returns healthy after model initialization.
- [ ] `MNEMOSYNE_LOCAL_EMBEDDING_URL` ending in `/v1` health-checks the service root `/health`, not `/v1/health`.
- [ ] Docker managed MinerU fails fast when `mineru:latest` is missing or NVIDIA runtime is not configured.

## MCP Smoke

- [ ] `knowcran serve-mcp-readonly` starts and exposes only readonly and audit tools.
- [ ] `knowcran serve-mcp-curate` starts and exposes curate tools.
- [ ] `knowcran serve-mcp-admin` starts and exposes admin tools.
- [ ] `tools/list` exposes declared tool schemas, required fields, and annotations.
- [ ] `tools/call knowcran_stats` succeeds against a temporary data directory.

## Safety

- [ ] Readonly profile cannot perform discovery, reading, review generation, export, repair, or dedupe.
- [ ] Data and vault directory escapes are rejected.
- [ ] Default tests require no live network, API key, or LLM provider.
- [ ] `.env.example` contains no secrets.

## Release

- [ ] Create Git tag `v1.0.0` and push.
- [ ] Verify `.github/workflows/release.yml` builds distribution, creates GitHub Release assets, and publishes to PyPI through Trusted Publishing.
- [ ] Configure the PyPI project Trusted Publisher for this repository, workflow file, and `pypi` environment before pushing the release tag.
- [ ] Verify release notes from `CHANGELOG.md` are copied into the GitHub Release.
- [ ] Verify `dist/SHA256SUMS.txt` is attached to the GitHub Release.
- [ ] Include a short migration note for users with pre-1.0 local databases.
